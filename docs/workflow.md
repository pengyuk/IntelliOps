# 应急流程与角色联动

## 目标

定义从告警到复盘的标准化流程、状态机、以及触达规则，减少人工沟通成本并保障责任闭环。

## 参与角色

- 值班运维（On-call）
- 开发负责人（Dev Owner）
- 业务联系人（PO/SM）
- 变更管理员
- 平台/安全负责人

## 流程步骤

1. 告警识别（自动）
   - 告警触发后根据规则自动归类并创建 `Incident`。
2. 初步评估（值班运维）
   - 填写`现场信息卡`：现象、影响、初步日志片段、时间线。
3. 紧急响应（按级别）
   - 轻度：由值班运维依据 SOP 处置并关闭。
   - 中度/严重：触发联动通知开发、业务、变更管理员。
4. 深入排查（协同）
   - Skill 引擎推荐排查路径，调用 KG 关联变更、依赖链路。
   - 选择自动化 Action 或人工步骤。
5. 缓解与恢复（记录）
   - 所有操作记录到工单/事件日志，必要时执行回滚/临时变更。
6. 复盘与改进
   - 生成复盘任务，指定责任人、截止日期，并写入知识库/案例库。

## 状态机（Incident）

- `New` -> `Investigating` -> `Mitigating` -> `Resolved` -> `Postmortem` -> `Closed`

## 通知与自动化规则

- 当 `Incident` 升级到 `Investigating` 时，自动通知 `Dev Owner` 和 `业务联系人`。
- 当检测到与最近变更高度相关（时间窗口/变更对象匹配），优先提示变更负责人。
- 复盘任务逾期自动催办并升学到管理层汇报。
