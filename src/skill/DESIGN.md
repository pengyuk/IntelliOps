# Skill 设计

## 目标

为运维/开发提供问答式与流程式交互能力，快速获得排查步骤、SOP 建议与 KG 关联视图。

## 组成

- NLU 层：意图识别（Incident 查询、排查建议、操作执行）
- 对话管理：基于场景的状态机（现场信息补录、步骤执行）
- 能力插件：KG 查询、推荐引擎、执行 Harness 接口
- 输出模块：文本建议、结构化卡片、可执行命令建议

## 典型意图

- `query_incident`：获取某次故障的概要
- `suggest_steps`：给出下一步排查建议
- `exec_action`：请求执行自动化脚本（权限校验）
- `attach_evidence`：上传日志/截图到事件

## 集成要点

- Skill 调用 KG 做上下文补全
- 输出应包含可操作项（一步建议 + 推荐的脚本/工单模板）
- 严格权限检查：`exec_action` 需二次确认与审计
