# War Room 角色矩阵

## 可用角色

| 角色 | 权限 | 典型人员 |
|------|------|---------|
| `operator` | 查看事故、执行操作、发送讨论 | 值班运维、SRE |
| `developer` | 查看事故、发送讨论、审查代码 | 应用开发、架构师 |
| `approver` | 查看事故、审批高风险操作、发布复盘 | 技术经理、TL |
| `admin` | 全部权限 | 系统管理员 |

## 通知规则

### 按故障类型

| 故障关键词 | 必须通知角色 | 建议通知角色 | 原因 |
|-----------|------------|------------|------|
| 数据库、DB、SQL、慢查询 | `operator` | `developer`, `approver` | DBA 或熟悉数据库的开发 |
| 网络、延迟、超时、丢包 | `operator` | `developer` | 网络团队 |
| OOM、CPU、内存、磁盘 | `operator` | `developer` | 基础设施/运维 |
| 部署、变更、配置、回滚 | `operator`, `developer` | `approver` | 变更相关方 |
| 安全、攻击、漏洞、泄露 | `operator`, `approver` | - | 安全事件需管理审批 |
| 支付、订单、交易 | `operator`, `developer` | - | 核心业务 |

### 按严重级别

| 级别 | 通知范围 | 升级条件 |
|------|---------|---------|
| P3（低） | 值班运维 | 30min 未解决 → 通知开发 |
| P2（中） | 运维 + 相关开发 | 1h 未解决 → 通知技术经理 |
| P1（高） | 运维 + 开发 + 技术经理 | 2h 未解决 → 通知 CTO |
| P0（紧急） | 全团队 + 管理层 | 立即通知 |

## 消息模板

### 故障通告

```
@all 故障通告：{{incident_id}} - {{故障摘要}}
影响范围：{{受影响服务}}
当前状态：{{Investigating/Mitigating/Resolved}}
负责人：{{值班运维}}
预计恢复：{{ETA}}
```

### 角色指派

```
@{{角色}} 请协助排查 {{具体事项}}
背景：{{故障简要描述}}
相关信息：{{诊断链接或关键发现}}
```

### 进度同步

```
📢 进度更新 {{incident_id}}
时间：{{当前时间}}
状态：{{当前状态}}
进展：{{最近完成的操作}}
下一步：{{待执行的操作}}
阻塞项：{{需要其他角色确认的事项}}
```

### 决策记录

```
📋 关键决策
决策：{{决策内容}}
提议人：{{提议人}}
确认人：{{确认人}}
理由：{{决策依据}}
时间：{{决策时间}}
```

## 通知方式

| 阶段 | 通知方式 | 说明 |
|------|---------|------|
| 原型 | `POST /incident/{id}/discussion` | 讨论区消息，所有在线用户可见 |
| 生产 | WebSocket 实时推送 | 接入后自动推送到在线用户 |
| 生产 | 企业微信/钉钉 Webhook | 离线用户通过 IM 接收 |
